# LOLA 2.0 — Что делать дальше

Ближайшие шаги по порядку. Опираемся на [задачи](LOLA-2.0-tasks.md) и [концепт](LOLA-2.0-concept.md).

---

## 1. Сейчас (этап подготовки)

### 1.1 Утвердить MVP-персонажей (5–10)
- **Готовый список по анализу запросов в пабликах:** [docs/LOLA-2.0-mvp-characters.md](LOLA-2.0-mvp-characters.md) — **10 персонажей:** ЧАПЛИН, ДЖИПСИ, ДОКТОР ЛАВ, МАЙЕВТИК, ЗВЕЗДНЫЙ ПРОВОДНИК, МАСТЕР ЧИСЕЛ, СКРУДЖ МАКДАК, МОРФЕЙ, ПРАНА, ШЕФ-ПОВАР.
- Можно заменить 1–2 по вкусу или добавить 11–12-м (например, МЕФИСТОФЕЛЬ, КОКО ШАНЕЛЬ).
- Зафиксировать итоговый список в концепте или оставить как в `LOLA-2.0-mvp-characters.md`.

### 1.2 Промпты для каждого персонажа
- Взять роль и стиль из скиллов (`.cursor/skills/<имя>/SKILL.md`).
- Оформить системный промпт под выбранный LLM (OpenAI, Claude, локальная модель и т.д.): 1 файл или 1 блок на персонажа.
- Хранить в репозитории (например, `prompts/` или в конфиге бэкенда).
- Для удобства использовать индивидуальные задачи обучения: [docs/LOLA-2.0-training-tasks.md](LOLA-2.0-training-tasks.md).

### 1.3 Выбрать стек
- **Мобильное приложение:** нативное (Swift/Kotlin), Flutter или React Native — с учётом команды и планов на iOS/Android.
- **Backend:** язык и фреймворк (Node, Python, Go и т.д.), хостинг (VPS, serverless).
- **LLM:** провайдер API, лимиты, стоимость за токен.
- Кратко записать решение в `docs/` (например, в конец концепта или в отдельный `tech-stack.md`).

### 1.4 Wireframes ключевых экранов
- Splash → Onboarding (2–4 экрана) → Main (персонажи) → Чат (поле + микрофон + Текст/Аудио) → Пейволл → Настройки.
- Достаточно чёрно-белых схем; можно в Figma, на бумаге или в макете в `docs/`.
- Сверять с [структурой экранов](LOLA-2.0-concept.md#3-структура-экранов) в концепте.

### 1.5 Юридическое
- Черновик политики конфиденциальности и пользовательского соглашения (обязательно для публикации и для голоса/аудио).
- Позже — доработка с юристом.

## 2. Дальше (когда подготовка закрыта)

- **Backend:** по задачам 1.1–1.27 из [LOLA-2.0-tasks.md](LOLA-2.0-tasks.md) — auth, пользователи, лимиты, персонажи, диалоги, LLM, STT/TTS-слой, подписка (webhooks), safety.
- **Мобильное приложение:** по задачам 2.1–2.46 — навигация, онбординг, главный экран, чат, микрофон (STT), ответ текст/аудио (TTS), пейволл, настройки, QA-гейт.
- **Тестирование:** сценарии из концепта (первый запуск, бесплатный лимит, премиум, восстановление покупок).
- **Публикация:** тестовые сборки (TestFlight / внутренняя ссылка), затем App Store / Google Play.
