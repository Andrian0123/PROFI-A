# Синхронизация с Git и закрытие доступа к репозиторию

## 1. Всегда синхронизировать проект с Git

### Быстрый способ (рекомендуется)

Запускайте скрипт в корне проекта:

- **`Синхронизация_с_Git.bat`** — двойной клик или из командной строки.

Скрипт делает:

1. `git pull origin main` — подтягивает изменения с GitHub.
2. `git add -A` и `git commit` — фиксирует все ваши изменения с датой и временем.
3. `git push origin main` — отправляет коммиты на GitHub.

Запускайте его после каждой рабочей сессии или перед закрытием проекта.

### Ручная синхронизация

В папке проекта в терминале:

```bash
git pull origin main
git add -A
git status
git commit -m "Описание изменений"
git push origin main
```

### Если Git пишет «dubious ownership»

Один раз выполните в терминале (под вашим пользователем):

```bash
git config --global --add safe.directory E:/PROFI-A
```

После этого команды `git` в этой папке будут выполняться без ошибки.

## 2. Закрыть доступ: сделать репозиторий приватным

Чтобы **никто не видел** ваш проект на GitHub:

1. Откройте в браузере: **https://github.com/andrian0123/PROFI-A**
2. Вкладка **Settings** (Настройки) репозитория.
3. Внизу страницы найдите блок **Danger Zone**.
4. Нажмите **Change repository visibility** → **Make private**.
5. Подтвердите, введя название репозитория `andrian0123/PROFI-A`.

После этого:

- Репозиторий видят только вы и добавленные вами **Collaborators**.
- Поиск и просмотр кода посторонними невозможны.
- Push и pull по-прежнему работают (нужна авторизация в GitHub при `git push`).

### GitHub Pages при приватном репозитории

- У приватного репозитория можно оставить GitHub Pages: сайт будет доступен по адресу вида `https://andrian0123.github.io/PROFI-A/`.
- Если **не хотите**, чтобы сайт был доступен публично: в **Settings → Pages** отключите источник (Source) или выберите «None».

### Если нужна авторизация при push

При первом `git push` после перевода в private GitHub может запросить логин/пароль. Рекомендуется использовать **Personal Access Token** вместо пароля:

1. GitHub → **Settings** (вашего аккаунта) → **Developer settings** → **Personal access tokens**.
2. Создайте токен с правом `repo`.
3. При запросе пароля введите этот токен.

| Задача | Действие |
|--------|----------|
| Регулярно синхронизировать код | Запускать **`Синхронизация_с_Git.bat`** или вручную делать pull → commit → push |
