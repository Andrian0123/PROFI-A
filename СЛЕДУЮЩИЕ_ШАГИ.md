1. Запустите **ОТКРЫТЬ_ЛОКАЛЬНЫЙ_СЕРВЕР.bat** (двойной щелчок). Окно не закрывайте.

3. Проверьте: экран входа — регистрация/вход (любой логин и пароль), раздел «Тех. поддержка» — отправка обращения. Должно работать через локальный сервер.

## Шаг 2. (По желанию) Деплой backend в Railway

Подробно по шагам: **ДЕПЛОЙ_ПО_ШАГАМ.md**. Кратко:

1. Откройте [railway.app](https://railway.app), войдите в аккаунт.
2. Нажмите **«+ Новое»** → **Deploy from GitHub repo**.
3. Выберите репозиторий **Andrian0123/PROFI-A**.
4. В настройках сервиса укажите:
   - **Root Directory:** оставьте пустым.
   - **Start Command:** `node server/server-single.js`
   - **Build Command:** пусто или `echo "No build"`
5. В **Settings → Networking** включите **Generate Domain** и скопируйте URL (например `https://profia-backend.up.railway.app`).
6. В `app/build.gradle.kts` в блоке `defaultConfig` замените три URL на этот один (без слэша в конце):
   - `AUTH_SERVER_URL`, `SUPPORT_SERVER_URL`, `SCAN_SERVER_URL` = один и тот же HTTPS-URL.
7. Пересоберите приложение. Тогда приложение будет ходить в облачный backend без запуска ПК.
