# Ссылка не работает (404) — как исправить

Страница «There isn't a GitHub Pages site here» появляется, пока в репозитории не включён **источник** для GitHub Pages и не выполнен деплой.

---

## Шаг 1. Включить GitHub Pages

1. Откройте в браузере: **https://github.com/Andrian0123/PROFI-A**
2. Вверху вкладки: **Settings**
3. Слева в меню: **Pages**
4. В блоке **Build and deployment**:
   - Поле **Source**: выберите **GitHub Actions** (не «Deploy from a branch»)
5. Ничего больше сохранять не нужно — выбор источника достаточно.

---

## Шаг 2. Запустить деплой (если сайт всё ещё 404)

1. В том же репозитории откройте вкладку **Actions**
2. Слева выберите workflow **Deploy site to GitHub Pages**
3. Справа нажмите **Run workflow** → снова **Run workflow**
4. Подождите 1–2 минуты, пока запуск станет зелёным (галочка)

---

## Шаг 3. Проверить ссылку

Откройте в браузере:

**https://andrian0123.github.io/PROFI-A/**

Должна открыться главная страница ПРОФЙ-А. Если снова 404 — подождите ещё 1–2 минуты и обновите страницу.

---

## Если Pages уже был включён

Зайдите в **Actions** и посмотрите последний запуск **Deploy site to GitHub Pages**:
- Если он **красный** (failed) — откройте его и посмотрите сообщение об ошибке в логе.
- Если **зелёный** — деплой прошёл; проверьте адрес ещё раз (иногда GitHub отдаёт страницу с небольшой задержкой).
