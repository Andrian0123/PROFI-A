# ПРОФЙ-А — Всё пошагово

Один документ со всеми шагами: от проверки сайта до публикации в интернет.

---

## Часть 1. Что уже создано

В папке **`E:\PROFI-A\site`** лежат готовые файлы:

| Файл | Что это |
|------|--------|
| index.html | Главная страница (название, «в разработке», описание, возможности, кнопка скачать, ссылки) |
| privacy.html | Политика конфиденциальности |

## Часть 2. Шаг за шагом — проверка на компьютере

### Шаг 1. Открыть папку сайта
1. Нажмите **Win + E** (проводник).
2. В адресной строке введите: **`E:\PROFI-A\site`** и Enter.
3. Убедитесь, что видите файлы **index.html**, **privacy.html**, **terms.html**.

### Шаг 2. Открыть главную страницу в браузере
1. В той же папке дважды щёлкните по **index.html**.
2. Должна открыться страница с названием «ПРОФЙ-А», блоком «Приложение ещё в разработке», описанием и кнопкой «Скачать в Google Play».

### Шаг 3. Проверить переходы
1. Внизу страницы нажмите **«Политика конфиденциальности»** — должна открыться страница с полным текстом политики.
2. Нажмите **«На главную»** — вернётесь на главную.
3. Нажмите **«Пользовательское соглашение»** — откроется страница с полным текстом соглашения.
4. Снова **«На главную»**.

Если всё открывается — сайт готов к публикации.

## Часть 3. Шаг за шагом — публикация (выберите один способ)

---

### Способ А. Публикация на GitHub Pages (автоматически после push)

Подходит, если проект уже на GitHub или вы готовы залить его туда.

**Шаг А1.** Включить Pages в репозитории  
1. Откройте репозиторий на **github.com**.  
2. Вкладка **Settings** → слева **Pages**.  
3. В блоке **Build and deployment** → **Source** выберите **GitHub Actions**.

**Шаг А2.** Отправить код на GitHub  
1. В Cursor или в папке `E:\PROFI-A` откройте терминал.  
2. Выполните по очереди:
   ```
   git add site .github/workflows/deploy-site.yml
   git commit -m "Сайт ПРОФЙ-А"
   git push origin main
   ```
   Если ветка называется не `main`, а `master`, замените `main` на `master`.

**Шаг А3.** Дождаться деплоя  
1. На GitHub откройте вкладку **Actions**.  
2. Должен запуститься workflow **Deploy site to GitHub Pages**.  
3. Дождитесь зелёной галочки (1–2 минуты).

**Шаг А4.** Открыть сайт  
1. Снова **Settings** → **Pages**.  
2. Вверху будет написано: «Your site is live at …».  
3. Ссылка вида: **https://ВАШ_ЛОГИН.github.io/PROFI-A/**  
4. Откройте её в браузере — это и есть ваш опубликованный сайт.

**Шаг А5.** Ссылки для магазинов приложений  
- Политика конфиденциальности: **https://ВАШ_ЛОГИН.github.io/PROFI-A/privacy.html**  
- Пользовательское соглашение: **https://ВАШ_ЛОГИН.github.io/PROFI-A/terms.html**  

Имя репозитория может отличаться — смотрите точный адрес в Settings → Pages.

### Способ Б. Публикация на Tilda (без хостинга)

Подходит, если вы хотите страницу на Tilda.

**Шаг Б1.** Открыть код для Tilda  
1. В папке **`E:\PROFI-A\site`** найдите файл **tilda-block.html**.  
2. Правый щелчок по нему → **Открыть с помощью** → **Блокнот**.

**Шаг Б2.** Скопировать весь код  
1. В Блокноте нажмите **Ctrl + A** (выделить всё).  
2. Нажмите **Ctrl + C** (скопировать).

**Шаг Б3.** Создать страницу на Tilda  
1. Зайдите на **tilda.cc** и войдите в аккаунт.  
2. Выберите сайт или создайте новый.  
3. Нажмите **«Создать страницу»** → **«С нуля»**.

**Шаг Б4.** Вставить блок с HTML  
1. На странице нажмите **«Добавить блок»** (плюс).  
2. Откройте раздел **«Другое»**.  
3. Выберите блок **«HTML-код»**.  
4. В поле для кода нажмите **Ctrl + V** (вставить).  
5. Сохраните блок (галочка или «Готово»).

**Шаг Б5.** Фон страницы (по желанию)  
1. Настройки страницы (шестерёнка) → **Фон**.  
2. Цвет фона: **#FFF8E7**.

**Шаг Б6.** Опубликовать  
1. Нажмите **«Опубликовать»**.  
2. Откройте ссылку на страницу — готово.

Политику и соглашение можно потом добавить отдельными страницами на Tilda и заменить в блоке ссылки `#privacy` и `#terms` на эти страницы.

### Способ В. Публикация на своём хостинге (reg.ru, timeweb, beget и т.п.)

**Шаг В1.** Получить файлы  
1. Откройте папку **`E:\PROFI-A\site`**.  
2. Скопируйте файлы: **index.html**, **privacy.html**, **terms.html** (и при желании **profi-a.html**).  
   Либо распакуйте **profi-a-site.zip** и возьмите файлы из распакованной папки.

**Шаг В2.** Зайти в панель хостинга  
1. Войдите в личный кабинет вашего хостинга.  
2. Откройте **«Файловый менеджер»** или **«Файлы»** (часто раздел «Файлы» / «Домен»).

**Шаг В3.** Открыть корень сайта  
1. Перейдите в папку **public_html** или **www** или **htdocs** (для вашего домена).  
2. Это и есть «корень» сайта — сюда загружаем файлы.

**Шаг В4.** Загрузить файлы  
1. Нажмите **«Загрузить»** / **Upload**.  
2. Выберите **index.html**, **privacy.html**, **terms.html** (и при желании **profi-a.html**).  
3. Дождитесь окончания загрузки.  
Важно: загружать именно **файлы**, а не папку с именем «site».

**Шаг В5.** Проверить  
1. Откройте в браузере адрес вашего сайта (например **https://ваш-домен.ru**).  
2. Должна открыться главная страница ПРОФЙ-А.  
3. Проверьте ссылки «Политика конфиденциальности» и «Пользовательское соглашение».

**Шаг В6.** Ссылки для магазинов  
- Политика: **https://ваш-домен.ru/privacy.html**  
- Соглашение: **https://ваш-домен.ru/terms.html**

## Часть 4. После публикации

1. **Карточка приложения (Google Play, RuStore)**  
   Укажите там ссылки на политику конфиденциальности и пользовательское соглашение (те, что получились в выбранном способе).

2. **Кнопка «Скачать в Google Play»**  
   Когда приложение будет опубликовано в магазине, замените в **index.html** и **profi-a.html** ссылку на кнопке на реальную ссылку на приложение в Google Play (и при желании добавьте кнопку RuStore).

3. **Надпись «Приложение ещё в разработке»**  
   Когда приложение выйдет, можно удалить этот блок из **index.html** (и из **tilda-block.html**, если используете Tilda).
