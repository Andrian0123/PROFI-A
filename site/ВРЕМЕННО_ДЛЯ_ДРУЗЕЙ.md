# Временно для друзей: собрать, выложить, отправить ссылку

Пока приложение не в Google Play (документы не готовы), можно раздавать APK по ссылке. Друзья открывают страницу на телефоне, нажимают «Скачать приложение (APK)», устанавливают. Позже опубликуете в Google Play и обновите кнопку на странице.

---

## Шаг 1. Собрать APK

В папке проекта `E:\PROFI-A` откройте терминал и выполните:

**Вариант A — Debug (быстро, для друзей):**
```bash
gradlew.bat assembleDebug
```
APK будет в папке: `app\build\outputs\apk\debug\app-debug.apk`

**Вариант B — Release (подписанный, для раздачи):**  
Нужна подпись (keystore). Если уже настроен release:
```bash
gradlew.bat assembleRelease
```
APK: `app\build\outputs\apk\release\app-release.apk`

Дальше в инструкции считаем, что вы берёте **app-debug.apk** (или переименуете release в тот же имя).

---

## Шаг 2. Положить APK на сайт

1. Скопируйте собранный APK (например `app-debug.apk`) в папку **`site/`**.
2. Переименуйте файл в **`profi-a.apk`** (именно так указано на странице).
3. Закоммитьте и отправьте на GitHub:
   ```bash
   git add site/profi-a.apk
   git commit -m "Add APK for temporary distribution"
   git push origin main
   ```
4. Подождите 1–2 минуты, пока обновится GitHub Pages.

После этого ссылка **https://andrian0123.github.io/PROFI-A/profi-a.apk** начнёт отдавать файл, а кнопка «Скачать приложение (APK)» на странице будет его скачивать.

---

## Шаг 3. Отправить ссылку друзьям

Отправьте друзьям ссылку на **страницу** (удобнее, чем сразу на APK):

**https://andrian0123.github.io/PROFI-A/**

Они откроют её на телефоне, нажмут «Скачать приложение (APK)», скачают файл и установят. На Android при установке из браузера может потребоваться разрешение «Установка из неизвестных источников» для браузера или «Файлы».

---

## Шаг 4. Когда опубликуете в Google Play

1. В **index.html** и **profi-a.html** замените кнопку:
   - Было: `href="profi-a.apk"` и текст «Скачать приложение (APK)»
   - Стало: `href="https://play.google.com/store/apps/details?id=ru.profia.app"` и текст «Скачать в Google Play»
2. Подпись под кнопкой можно заменить на: «Приложение для Android».
3. В **tilda-block.html** (если используете Tilda) замените ссылку и текст так же.
4. Файл **profi-a.apk** из папки `site/` можно удалить и закоммитить, чтобы не хранить старый APK в репозитории (по желанию).

---

## Важно

- **Debug APK** подходит для друзей и теста. Для публикации в магазине нужна **release**-сборка с вашей подписью.
- Размер APK (20–50 МБ и больше) — GitHub допускает файлы до 100 МБ. Если APK больше, используйте [GitHub Releases](https://github.com/Andrian0123/PROFI-A/releases): создайте релиз, прикрепите APK, в кнопке на странице укажите ссылку на файл из Releases.
