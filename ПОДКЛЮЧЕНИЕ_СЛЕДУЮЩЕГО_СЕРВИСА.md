## Следующий бесплатный сервис: облачный backend

Подключаем **один бесплатный хостинг** для Auth + Support + Scan, чтобы приложение работало без запущенного ПК и из любой сети.

### Вариант A — Railway (рекомендуется)

- **Тариф:** $5 кредитов/месяц (хватает на тест).
- **Шаги:** пошагово в **ДЕПЛОЙ_ПО_ШАГАМ.md** (4 шага).
- **Кратко:** [railway.app](https://railway.app) → Deploy from GitHub → репозиторий **Andrian0123/PROFI-A** → Start Command: `node server/server-single.js` → Generate Domain → скопировать HTTPS-URL → подставить в `app/build.gradle.kts` в три поля (AUTH, SUPPORT, SCAN) → пересобрать приложение.

### Вариант B — Render

- **Тариф:** Free tier (сервис засыпает после ~15 мин неактивности, первый запрос 30–60 с).
- **Шаги:** **docs/ПОШАГОВОЕ_ПОДКЛЮЧЕНИЕ_СЕРВИСОВ.md** — раздел «Шаг 4. Деплой в облако Render».
- **Кратко:** [render.com](https://render.com) → New → Web Service → репозиторий ПРОФЙ-А → Runtime: Node → Start Command: `node server/server-single.js` → Instance Type: Free → создать → скопировать URL → прописать в `build.gradle.kts` так же, как для Railway.

После подключения одного из вариантов приложение будет ходить в облако по HTTPS; локальный BAT не нужен.
