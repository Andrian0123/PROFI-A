# Пошаговый путь регистрации сервисов и ссылки

Список сервисов PROFI-A с пошаговой инструкцией регистрации и прямыми ссылками.

---

## 1. Google Play (публикация приложения и подписки)

### 1.1 Аккаунт разработчика Google Play

**Ссылка для регистрации:**  
https://play.google.com/console/signup  

**Пошагово:**

1. Откройте ссылку в браузере, войдите в Google-аккаунт (или создайте новый).
2. Примите **Developer Distribution Agreement** (соглашение разработчика).
3. Оплатите разовый взнос **$25** (карта Visa, Mastercard, American Express и др.; предоплаченные карты не подходят).
4. Выберите тип аккаунта: **физическое лицо** или **организация** (для ИП/ООО).
5. Заполните данные профиля разработчика (имя, страна, контакты).
6. При необходимости пройдите верификацию личности (документ, удостоверяющий личность).
7. После активации аккаунта перейдите в [Play Console](https://play.google.com/console) и создайте первое приложение.

**Справка:**  
https://support.google.com/googleplay/android-developer/answer/6112435  

---

### 1.2 Настройка подписок (Billing)

Подписки настраиваются **внутри** Play Console (отдельной регистрации нет).

**Пошагово:**

1. Войдите в [Play Console](https://play.google.com/console).
2. Выберите приложение (или создайте его).
3. Раздел **«Монетизация»** → **«Продукты»** → **«Подписки»**.
4. Создайте подписки с Product ID: `profi_a_sub_1`, `profi_a_sub_6`, `profi_a_sub_12` (как в приложении).
5. Укажите цены и периоды (1, 6, 12 месяцев).

**Справка:**  
https://support.google.com/googleplay/android-developer/answer/1217309

## 2. Сервер (VPS) для бэкенда (Auth, Support, Scan, Verification)

### 2.1 Selectel (Россия, СНГ)

**Ссылка для регистрации:**  
https://selectel.ru/registration  

**Пошагово:**

1. Перейдите по ссылке.
2. Укажите **email** и задайте **пароль** (или используйте «Войти через соцсеть»).
3. Примите согласие на обработку персональных данных.
4. Подтвердите email по письму.
5. В панели управления: **«Облачная платформа»** → **«Серверы»** → **«Создать сервер»**.
6. Выберите образ (Ubuntu 22.04), конфигурацию (от 1 vCPU, 2 GB RAM), регион и создайте сервер.
7. Привяжите способ оплаты (карта или счёт для юрлиц).

**VPS/облако:**  
https://selectel.ru/services/cloud/servers/  

---

### 2.2 Timeweb Cloud (Россия)

**Ссылка для регистрации:**  
https://timeweb.cloud/?auth=registrationAccount  

**Пошагово:**

1. Откройте ссылку, нажмите **«Создать аккаунт»**.
2. Введите email и пароль, подтвердите регистрацию.
3. В панели: **«Серверы»** → **«Создать сервер»** (VPS/VDS).
4. Выберите ОС (Ubuntu), конфигурацию и регион (Москва, СПб и др.).
5. Оплата по факту использования или предоплатой.

**Тарифы VPS:**  
https://timeweb.cloud/services/vps-linux  

---

### 2.3 DigitalOcean (международный)

**Ссылка для регистрации:**  
https://cloud.digitalocean.com/registrations/new  

**Пошагово:**

1. Перейдите по ссылке.
2. Регистрация через **email**, **Google** или **GitHub**.
3. Добавьте способ оплаты (карта; часто есть промокод на кредиты).
4. В панели: **«Create»** → **「Droplets」** → выберите образ (Ubuntu), размер (от $5–6/мес), регион.
5. Создайте Droplet и подключайтесь по SSH.

**Документация:**  
https://docs.digitalocean.com/docs/droplets/

## 3. Бессерверные / managed-бэкенды (альтернатива своему VPS)

### 3.1 Firebase (Auth, БД, хостинг)

**Ссылка для регистрации:**  
https://console.firebase.google.com/  

**Пошагово:**

1. Войдите через Google-аккаунт.
2. **«Add project»** / **«Создать проект»** → введите название (например, `profia-app`).
3. При необходимости подключите Google Analytics (можно отключить).
4. После создания: **Authentication** → включите нужные методы (Email/Password и т.д.).
5. Для API своего бэкенда можно использовать **Cloud Functions** (нужен план Blaze при выходе за бесплатный лимит).

**Документация:**  
https://firebase.google.com/docs  

---

### 3.2 Supabase (Auth, БД, API)

**Ссылка для регистрации:**  
https://supabase.com/dashboard/sign-up  

**Пошагово:**

1. Перейдите по ссылке.
2. Регистрация через **GitHub** или **email + пароль** (пароль: буквы, цифры, спецсимвол, от 8 символов).
3. Создайте **New project**: укажите имя, пароль БД, регион.
4. В разделе **Authentication** настройте провайдеры (Email и др.).
5. В **Table Editor** создайте таблицы при необходимости; API генерируется автоматически.

**Документация:**  
https://supabase.com/docs

## 4. Домен (для API и сайта)

### 4.1 Reg.ru (Россия)

**Ссылка:**  
https://www.reg.ru/  

**Пошагово:**

1. Зарегистрируйтесь в личном кабинете (если ещё нет аккаунта).
2. На главной введите желаемое имя домена (например, `profia-api`) и нажмите **«Подобрать»**.
3. Выберите свободный домен (например, .ru, .com) и нажмите **«Перейти к заказу»**.
4. Заполните данные администратора домена, оплатите (аренда от 1 года).
5. В панели управления настройте DNS: A-запись на IP вашего VPS или CNAME на хостинг.

**Помощь по регистрации домена:**  
https://help.reg.ru/support/domains/registratsiya-domena/kak-zaregistrirovat-domen  

---

### 4.2 Namecheap (международный)

**Ссылка:**  
https://www.namecheap.com/  

**Пошагово:**

1. Создайте аккаунт (Sign Up).
2. Поиск домена → добавьте в корзину и оплатите.
3. В панели **Domain List** → **Manage** → **Advanced DNS** добавьте A-запись на IP сервера.

## 5. SSL (HTTPS)

**Let's Encrypt (бесплатно)** — отдельной регистрации нет, настраивается на вашем сервере.

**Пошагово (на VPS с Ubuntu):**

1. Установите certbot:  
   `sudo apt install certbot python3-certbot-nginx`  
   (или для другого веб-сервера — см. документацию certbot).
2. Выполните:  
   `sudo certbot --nginx -d yourdomain.com`  
   (подставьте ваш домен).
3. Следуйте подсказкам; сертификат продлевается автоматически.

**Ссылка:**  
https://letsencrypt.org/

## 7. Рекомендуемый порядок для старта

1. **Google Play** — зарегистрировать аккаунт разработчика ($25) и создать приложение.
2. **VPS** — выбрать один провайдер (Selectel / Timeweb / DigitalOcean), зарегистрироваться и создать сервер.
3. **Домен** (по желанию) — зарегистрировать домен и привязать к VPS через DNS.
4. **SSL** — установить certbot на VPS и получить сертификат для домена.
5. **Firebase или Supabase** — только если решите не поднимать свой Auth на VPS, а использовать managed-сервис.

После этого в `app/build.gradle.kts` укажите полученные URL (AUTH_SERVER_URL, SUPPORT_SERVER_URL, SCAN_SERVER_URL, VERIFICATION_SERVER_URL).
